Node Safety Check source ↗

Vet a ComfyUI custom node before you install it.

Static analysis for ComfyUI custom-node repos. Paste a GitHub URL, source, or upload files — get a risk score with line-level findings. Everything runs locally in your browser; no source is uploaded.

What this is and isn't. Static analysis catches patterns associated with documented attacks (credential stealers, cryptominers, drive-by downloaders). It cannot evaluate runtime behaviour or detect threats that obfuscate beyond simple patterns. Treat a clean result as a quick checkpoint, not a guarantee. For high-stakes installs, read the source yourself.

What gets checked

Patterns associated with documented ComfyUI custom-node attacks (LLMVISION credential stealer, Akira stealer in Upscaler_4K, cryptominer botnet campaigns):

  • Critical: obfuscated exec/eval chains (base64 → exec, chr-assembled strings), pickle.loads on untrusted input, marshal.loads, exfiltration of browser profile / SSH / keychain data, writes to OS autostart paths, hardcoded crypto-wallet addresses.
  • High: os.system, subprocess with shell=True + variable input, ctypes native DLL loading, runtime pip install, importlib with variable arg.
  • Medium: subprocess without shell, raw socket use, network calls in setup.py, URLs to non-mainstream hosts (pastebin, discord webhooks, raw IPs).
  • Info: unpinned dependencies, GitHub-URL dependencies (rug-pull risk), requests/urllib imports.

Limits

Static analysis cannot evaluate runtime behaviour. Specifically: code that fetches a payload at runtime, evaluates strings reconstructed via arithmetic, or hides logic in a compiled .pyc bundled with the node will not be caught. The official ComfyUI security review (private Discord channel) and manual code review remain the gold standards.

Why this exists

The Comfy-Spaces safety scanner proposal has been open since January 2026 with no shipped browser-based tool. Multiple real attacks have been published (LLMVISION, cryptominer botnet, Upscaler_4K + Akira stealer). This page fills the "I want a quick check before I install this random node" gap that CLI-only tools (christian-byrne/custom-nodes-security-scan) don't reach.
